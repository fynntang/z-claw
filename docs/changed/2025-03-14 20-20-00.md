# 变更记录 2025-03-14 20:20:00

## 概述

补全设置页配置项：除「提供商」外，增加请求超时、Agent、记忆、自主性等与 ZeroClaw Config 一致的设置。

## 修改内容

### 1. 后端 `src-tauri/src/bridge/config.rs`

- **AppConfig 扩展**：
  - 请求与超时：`provider_timeout_secs`（默认 120）、`api_path`（可选）
  - Agent：`max_tool_iterations`、`max_history_messages`、`compact_context`、`parallel_tools`
  - 记忆：`memory_backend`、`memory_auto_save`
  - 自主性：`autonomy_level`（"readonly" | "supervised" | "full"）
- **get_config()**：从 zeroclaw::Config 读出上述字段填入 AppConfig。
- **set_config()**：将上述字段写回 zeroclaw_config（agent、memory、autonomy、provider_timeout_secs、api_path）。
- **validate_config()**：增加 provider_timeout_secs、max_tool_iterations、max_history_messages > 0 的校验。
- **辅助**：`autonomy_level_to_string`、`parse_autonomy_level` 与 zeroclaw::AutonomyLevel 互转。
- **测试**：test_validate_config_valid 使用 `..AppConfig::default()` 以包含新字段。

### 2. 前端 `src/lib/components/Settings.svelte`

- **ConfigState 与默认值**：增加与 AppConfig 一致的字段及默认值（对齐 zeroclaw 默认）。
- **UI 分组**：
  - **请求与超时**：API 超时（秒）、API 路径后缀（可选）。
  - **Agent**：最大工具调用次数、最大历史消息数、compact_context、parallel_tools 勾选。
  - **记忆**：后端下拉（sqlite/markdown/none/lucid/postgres/qdrant）、auto_save 勾选。
  - **自主性**：级别下拉（readonly / supervised / full）及简短说明。
- **图标**：Timer、Bot、Database、Shield 用于各区块标题。

## 技术说明

- 自主性 serde 使用小写 "readonly"、"supervised"、"full"；前端与后端一致。
- 记忆后端选项与 schema 中 MemoryConfig 支持一致（sqlite 默认）。
- 设置页与 zeroclaw::Config 默认值对齐，保存时整份 AppConfig 写入 config.toml。

## 验证

- `cargo check --manifest-path src-tauri/Cargo.toml` 通过。
- 前端加载/保存应包含所有新字段；保存后 config.toml 中应有 [agent]、[memory]、[autonomy] 等。
