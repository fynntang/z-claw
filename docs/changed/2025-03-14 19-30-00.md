# 变更记录 2025-03-14 19:30:00

## 概述

新增「Agent 详情」与「Workspace 文件」右侧面板：展示 ZeroClaw workspace 下的 IDENTITY.md、USER.md 等文件内容，以及 workspace 目录树。

## 修改内容

### 1. 后端 (src-tauri)

- **bridge/workspace.rs（新增）**
  - `workspace_path()`：返回 workspace 根路径字符串（与 config 一致：`config_dir/zeroclaw/workspace`）。
  - `workspace_list_dir(subpath)`：列出指定子路径下的条目（name, is_dir），subpath 为空或 null 表示根目录。
  - `workspace_read_file(relative_path)`：读取工作区内相对路径文件内容（如 IDENTITY.md、USER.md、memory/xxx.md）。
  - 路径校验：禁止 `..` 与绝对路径，相对路径统一为正斜杠。
- **lib.rs**：注册 Tauri commands：`workspace_path`、`workspace_list_dir`、`workspace_read_file`。
- **bridge/mod.rs**：导出 `workspace` 模块及 `WorkspaceEntry`。

### 2. 前端 (src/routes/+page.svelte)

- **右侧面板模式**：由单一「工具与状态」改为三种模式切换：
  - **工具与状态**：原有运行时信息 + 工具列表。
  - **Agent 详情**：展示 workspace 下的 IDENTITY.md、USER.md、SOUL.md、AGENTS.md、MEMORY.md；Tab 切换，内容为只读文本。
  - **Workspace 文件**：展示 workspace 根路径及目录树；根目录列表由 `workspace_list_dir('')` 获取，点击目录可展开一层子项（再调 `workspace_list_dir(subpath)`）。
- **顶部按钮**：增加「Agent 详情」「Workspace」两个按钮，与「工具与状态」并列，点击切换右侧面板内容或关闭面板。

## 技术说明

- Workspace 路径与 ZeroClaw 一致：`dirs::config_dir().join("zeroclaw").join("workspace")`（如 Windows 为 `%APPDATA%\zeroclaw\workspace`，Linux 为 `~/.config/zeroclaw/workspace`）。若使用 `~/.zeroclaw/workspace` 需在 ZeroClaw 侧通过环境变量或配置指定。
- Agent 详情在打开面板时一次性请求上述 5 个文件；缺失文件显示「[文件不存在或无法读取]」。
- Workspace 目录树仅展开一层子目录，未做多级递归加载。
