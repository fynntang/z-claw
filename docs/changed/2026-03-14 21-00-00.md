# 变更记录 2026-03-14 21:00:00

## 概述

从「git submodule + path 依赖 zeroclaw 库」改为「嵌入 zeroclaw 二进制 + 通过 Gateway HTTP API 接入」。应用启动时拉起 zeroclaw gateway 进程，所有配置/状态/工具列表经 REST API 完成。

## 修改内容

### 1. 依赖与构建

- **Cargo.toml**：workspace `members` 改为仅 `["src-tauri"]`，移除 `workspace.dependencies.zeroclaw`。
- **src-tauri/Cargo.toml**：移除 `zeroclaw` 依赖，新增 `tauri-plugin-shell`。
- **crates/zeroclaw**：删除原 stub 的 `Cargo.toml` 与 `src/lib.rs`，保留目录并添加 `.gitkeep`；用于可选「从源码编译」时通过 submodule 填充。
- **.gitmodules**：新增，`crates/zeroclaw` 指向 `https://github.com/zeroclaw-labs/zeroclaw.git`，便于 `ZEROCLAW_BUILD_FROM_SOURCE=1` 时本地编译。

### 2. 二进制获取（默认 Release + 可选本地编译）

- **scripts/ensure-zeroclaw-binary.ps1**：Windows 下按 `CARGO_BUILD_TARGET` 或 rustc 获取 target，优先从 GitHub Release（默认 v0.1.7）下载对应 asset，解压到 `src-tauri/binaries/` 并命名为 Tauri externalBin 约定（如 `zeroclaw-x86_64-pc-windows-msvc.exe`）；失败或 `ZEROCLAW_BUILD_FROM_SOURCE=1` 时在 `crates/zeroclaw` 执行 `cargo build --release --bin zeroclaw` 并拷贝产物。
- **scripts/ensure-zeroclaw-binary.sh**：Unix 下逻辑同上。
- **scripts/ensure-zeroclaw.js**：跨平台入口，按平台调用上述脚本；`package.json` 中 `build` 改为先执行 `pnpm run ensure-zeroclaw` 再 `vite build`。
- **tauri.conf.json**：`bundle.externalBin` 增加 `"binaries/zeroclaw"`。

### 3. Gateway 管理与 API 客户端

- **src-tauri/src/gateway.rs**：Gateway 状态（端口、token）、spawn sidecar（`zeroclaw gateway --host 127.0.0.1 --port 42617`）、轮询 `/health`、配对（`/admin/paircode` 或 `/admin/paircode/new` + `POST /pair`）、token 持久化（`~/.config/zeroclaw/desktop-token`）、以及 `api_status`、`api_config_get`、`api_config_put`、`api_tools` 等 HTTP 封装。
- **src-tauri/src/config_bridge.rs**：将 Gateway 返回的 TOML 字符串与前端 `ConfigState` 互转（`toml_to_config_state`、`merge_config_state_into_toml`）。

### 4. Tauri 启动与 Commands

- **src-tauri/src/lib.rs**：`setup` 中注册 `AppGatewayState`，异步执行 `ensure_gateway_ready`（spawn sidecar、等 health、配对并写 token）。注册命令：`get_status`、`config_get`、`config_set`、`config_validate`、`tools_list`、`providers_list`、`workspace_path`、`workspace_read_file`、`workspace_list_dir`、`chat`；其中 status/config/tools 经 Gateway API 实现，workspace/chat 为占位。
- **capabilities/default.json**：增加 `shell:allow-execute`、`shell:allow-spawn`，用于 sidecar 启动。

### 5. 行为说明

- 桌面端默认在本机 spawn zeroclaw gateway（端口 42617），并通过 `/api/*` 与配对 token 访问；可选环境变量 `ZEROCLAW_USE_EXTERNAL_GATEWAY=1` 时不 spawn，仅作 HTTP 客户端（需本机已有 gateway 与已配对 token）。
- 首次运行若无 token，会自动请求 paircode 并完成配对，token 写入 `~/.config/zeroclaw/desktop-token`（或 Windows 等价目录）。

## 验证

- `cargo check --manifest-path src-tauri/Cargo.toml` 通过。
- `node scripts/ensure-zeroclaw.js` 可下载 zeroclaw 二进制至 `src-tauri/binaries/`。
