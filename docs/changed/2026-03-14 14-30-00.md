## 修改摘要

### 1. `src/lib/stores/app.ts`
- **AgentResponse**：新增类型，与 Rust 端 `AgentResponse` 一致（content, success, error?, session_id?）。
- **sendChat(message)**：调用 Tauri `chat` 命令，从 store 读取 `config`、`currentSessionId`，组装 snake_case 入参（session_id, api_key, api_url, provider, model），返回 AgentResponse。
- **normalizeBackendConfig**：后端 `config_get` 返回 snake_case，转为前端 camelCase 写入 config store，保证全站配置键统一。
- **toBackendConfig**：前端保存/校验配置时，将 camelCase 转为 snake_case 再传给 `config_set` / `config_validate`。
- **loadConfig**：使用 `normalizeBackendConfig` 处理返回值。
- **saveConfig / validateConfig**：使用 `toBackendConfig` 构建入参。
- 引入 `get` from 'svelte/store' 用于 sendChat 内读取 config 与 currentSessionId。

### 2. `src/lib/components/ChatArea.svelte`
- 使用 store：`messages`、`currentSessionId`、`sendChat`、`generateId`、`getCurrentTime`。
- **inputText**、**sending**：本地状态，绑定输入框与发送中态。
- **handleSend**：校验非空、防重复；追加 user 消息 → 清空输入 → 调用 `sendChat` → 根据返回设置 `currentSessionId`、追加 assistant 消息（成功内容或错误文案）；catch 时追加“请求异常”消息。
- **onTextareaKeydown**：Enter 无 Shift 时提交，阻止默认换行。
- 消息列表改为渲染 `$messages`，发送中显示“正在回复...”占位。
- 发送按钮与 textarea 在 sending 时 disabled。

### 3. `src/routes/+layout.svelte`
- **onMount** 中调用 `loadConfig()`，保证应用启动后配置可用，供 Chat 使用。

### 4. `src/lib/components/TitleBar.svelte`
- **runtimeStatus**：onMount 时调用 Tauri `get_status`，保存 `{ provider, model }`。
- 标题栏中部展示：若 provider/model 非 "none" 则显示 `provider · model`，否则显示“未就绪”（与当前 DesktopBridge 返回一致）。
