# 变更记录 2025-03-14 20:35:00

## 概述

桌面端不再直接依赖 zeroclaw 内部 API（不修改 zeroclaw 项目）。配置与工作区路径、自主性等均在桌面端本地实现或通过 TOML 读写。

## 修改内容

### 1. zeroclaw（还原对外暴露）

- **crates/zeroclaw/src/lib.rs**：移除 `pub use security::AutonomyLevel`。zeroclaw 保持为上游/未管理依赖时，桌面端不依赖其内部类型。

### 2. 桌面端 config 桥接（src-tauri/src/bridge/config.rs）

- **路径**：不再调用 `zeroclaw::config::schema::default_config_and_workspace_dirs()`，改为本地 `get_zeroclaw_dirs_local()`，使用 `dirs::home_dir().join(".zeroclaw")` 与 `config_dir.join("workspace")`，与 zeroclaw 约定一致。
- **读取 autonomy**：从 config 文件解析为 `toml::Value`，读取 `["autonomy"]["level"]` 字符串，不再使用 `zeroclaw::AutonomyLevel`。
- **写入配置**：保存时不再反序列化/序列化 `zeroclaw::Config`，改为读取为 `toml::Value`，仅更新 GUI 相关键（顶层、[agent]、[memory]、[autonomy]），再写回文件，从而不依赖 `zeroclaw::AutonomyLevel`。
- **默认值**：`AppConfig::default()` 中 `autonomy_level` 固定为 `"supervised"`，不再从 `c.autonomy.level` 转换。
- 删除 `autonomy_level_to_string`、`parse_autonomy_level`。

### 3. 桌面端 workspace 桥接（src-tauri/src/bridge/workspace.rs）

- **路径**：不再使用 `zeroclaw::config::schema::default_config_and_workspace_dirs()`，改为 `dirs::home_dir().map(|h| h.join(".zeroclaw").join("workspace"))`，与 zeroclaw 约定一致。

## 技术说明

- zeroclaw 的 [autonomy] level 使用小写：`readonly` | `supervised` | `full`；桌面端写入时做相同规范化。
- 桌面端仍依赖 `zeroclaw::Config` 用于：解析 config.toml（get_config 的其余字段）、`AppConfig::default()` 的默认值来源。不再依赖 `zeroclaw::config::schema` 或 `zeroclaw::AutonomyLevel`。
