# 变更记录 2025-03-14 20:00:00

## 概述

1. Workspace 目录改为通过 zeroclaw 的 `default_config_and_workspace_dirs()` 获取；配置路径与 zeroclaw 一致（~/.zeroclaw）。
2. 设置页配置项与 zeroclaw `Config` 结构及默认值对齐（参考 schema.rs Default、wizard 构建方式）。

## 修改内容

### 1. zeroclaw (crates/zeroclaw)

- **config/schema.rs**：将 `default_config_and_workspace_dirs()` 改为 `pub`，并补充文档说明供 desktop 等调用方同步解析默认 (config_dir, workspace_dir)（~/.zeroclaw、~/.zeroclaw/workspace）。

### 2. 后端 bridge (src-tauri)

- **bridge/workspace.rs**：不再使用 `dirs::config_dir().join("zeroclaw").join("workspace")`，改为调用 `zeroclaw::config::schema::default_config_and_workspace_dirs()` 取 workspace 根目录；失败时回退到 `./zeroclaw/workspace`。
- **bridge/config.rs**：
  - 路径：使用 `default_config_and_workspace_dirs()` 得到 (config_dir, workspace_dir)，config 文件为 `config_dir.join("config.toml")`（即 ~/.zeroclaw/config.toml）。
  - AppConfig 默认值：由 `zeroclaw::Config::default()` 推导（与 schema.rs 及 wizard 一致）—— provider = "openrouter"，model = "anthropic/claude-sonnet-4.6"，api_url 默认 "https://openrouter.ai/api/v1"，temperature = 0.7。
  - get_config()：从文件解析出 `zeroclaw::Config` 后，缺失字段用 `Config::default()` 的对应值填充再映射到 AppConfig。
  - 单元测试：预期默认 provider 为 openrouter，校验用例改为使用 openrouter URL 与 model。

## 技术说明

- 当前仅使用同步的 `default_config_and_workspace_dirs()`，未使用 `resolve_runtime_dirs_for_onboarding()`（会考虑 ZEROCLAW_WORKSPACE、active_workspace.toml 等），因此桌面端与 zeroclaw 默认目录布局一致（~/.zeroclaw、~/.zeroclaw/workspace）。
- 设置页 UI 仍只编辑 AppConfig 的 api_key、api_url、provider、model、temperature；与 zeroclaw::Config 的对应关系及默认值已统一。
