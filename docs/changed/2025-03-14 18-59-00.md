# 变更记录 2025-03-14 18:59:00

## 概述

按项目分析计划实现：设置页按 ZeroClaw 配置项重设计、主界面改为单 Agent UI。

## 修改内容

### 1. 设置页面 (`src/lib/components/Settings.svelte`)

- **重设计**：单面板表单，无多 Tab；仅保留与 ZeroClaw/AppConfig 对应项。
- **配置项**：提供商（下拉，来自 `providers_list`）、API 地址、API Key、默认模型、温度（0–2 滑块）。
- **移除**：General/Providers/Models/Network/Storage/Security/Advanced 等多 Tab、自定义 Provider（localStorage）、多 Provider 连接/断开等与当前后端不一致的 UI。
- **保存**：调用 `config_validate` 后 `config_set`，错误信息展示在页脚。
- **样式**：简洁卡片弹窗、清晰标签与间距、加载/保存状态反馈。

### 2. 主页面单 Agent UI (`src/routes/+page.svelte`)

- **左侧**：由宽侧栏改为窄左轨（约 64px），仅保留 ZClaw 标识 + 日志 + 配置按钮；移除 Agents/Sessions/IM Channels/Cron 等占位。
- **顶部**：单行标题「ZClaw」+ 当前 provider/model 文案 +「工具与状态」切换按钮。
- **中间**：保留原有聊天与发送逻辑；消息气泡与输入区样式简化，文案改为中文（如「思考中…」「输入消息，Enter 发送」）。
- **右侧**：仅保留「状态与工具」面板（可折叠），展示运行时状态（provider、model、状态）与工具列表；移除「Workspace / Project Files」占位。
- **欢迎语**：首条助手消息改为「你好，我是 ZClaw。有什么可以帮你的？」。

## 技术说明

- 设置页与后端 `bridge/config.rs` 中 `AppConfig` 一致：`api_key`、`api_url`、`provider`、`model`、`temperature`。
- 主页面不再使用 `rightPanelActive`（files/agent），改为单一布尔 `showToolsPanel` 控制右侧工具面板。
- 修复 Svelte 自闭合标签告警（`<div />` → `<div></div>` 等）。
