## 2. 技术栈

### 前端
- **框架**: SvelteKit 2.x + Svelte 5（使用 `$state` / `$derived` / `$props`）
- **构建**: Vite 7.x
- **样式**: Tailwind CSS 4.x、tw-animate-css、tailwind-variants
- **UI 组件**: bits-ui、layerchart、paneforge、vaul-svelte、svelte-sonner、Lucide 图标
- **表单**: sveltekit-superforms、formsnap
- **桌面壳**: Tauri 2.x（`@tauri-apps/api`）

### 后端（Tauri）
- **语言**: Rust (edition 2024)
- **运行时**: Tokio（async）
- **核心依赖**: ZeroClaw（path 依赖，位于 `crates/zeroclaw` 子模块）
- **配置**: `dirs` + `toml`，读写 `~/.zeroclaw/config.toml`
- **HTTP**: reqwest（rustls）

### 子模块
- **crates/zeroclaw**: ZeroClaw 上游（Rust 自主代理运行时），通过 Git 子模块引入，版本由子模块提交固定。

## 4. 架构要点

### 4.1 集成模式（ADR 0001）
- **嵌入式模式**：Tauri 直接调用 ZeroClaw 的 Rust 代码，不跑独立 daemon。
- **边界**：前端仅通过 Tauri 的 invoke 调用约定好的命令，不直接接触 ZeroClaw 内部。

### 4.2 Tauri 命令（已实现）
- `greet`：示例
- `chat`：与 AI 对话（支持 session_id、api_key、api_url、model 等）
- `get_status`：运行时状态
- `config_get` / `config_set` / `config_validate`：配置读写与校验
- `tools_list`：可用工具列表
- `logs_tail` / `logs_clear`：日志拉取与清空

配置与契约详见 `docs/contracts/tauri-command-contract-v1.md`（含 status/chat 等请求响应形态）。

### 4.3 配置与安全
- API Key 等敏感信息存于 `~/.zeroclaw/config.toml`，由后端读写，**不**写入前端 localStorage。
- 前端 `app.ts` 中的 `config` 仅内存态，通过 `loadConfig()` / `saveConfig()` 与后端同步。

### 4.4 前端数据流
- **Stores**（`src/lib/stores/app.ts`）：`messages`、`sessions`、`currentSessionId`、`agents`、`currentAgentId`、`config`、`configLoaded`、`configLoading`、`showSettings`、`showAgentInfo`。
- **页面**（`+page.svelte`）：左侧 Agent 列表（AgentSidebar）、频道/会话列表（ChannelSidebar），中间 ChatArea + 右侧 RightPanel（Agent/文件），全局 SettingsPanel。

## 5. 主要界面与组件

| 组件 | 职责 |
|------|------|
| **TitleBar** | 窗口标题栏（无边框窗口） |
| **AgentSidebar** | 多 Agent 切换（当前为苏东坡、李白、杜甫、王维等占位） |
| **ChannelSidebar** | 频道/会话列表（如 chat-1） |
| **ChatArea** | 当前 Agent 的聊天区、输入框、顶部「文件/Agent」切换 |
| **RightPanel** | 右侧面板（Agent 详情 / 文件），可折叠 |
| **SettingsPanel** | 设置（与 config 联动） |

当前 ChatArea 使用 `mockMessages`（多为空），界面为「等待输入中...」，真实对话需接 Tauri `chat` 命令与 stores。

## 8. 总结与建议

- **项目性质**：ZeroClaw 的桌面壳，前端负责会话/多 Agent/频道/设置展示与交互，后端负责配置、对话、状态、工具、日志，通过 Tauri 命令与嵌入式 ZeroClaw 通信。
- **可执行性**：当前结构可执行（`pnpm tauri dev`）、可验证（cargo check + 前端 check）、可维护（ADR + 契约 + 子模块固定版本）。
- **建议**：  
  1）将 ChatArea 与 Tauri `chat`、`get_status` 及 stores 打通，替换 mock 数据；  
  2）新功能或命令变更时同步更新 `docs/contracts/tauri-command-contract-v1.md`；  
  3）保持 `crates/zeroclaw` 子模块按需更新并显式提交子模块指针，避免随意拷贝代码。
